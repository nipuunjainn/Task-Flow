# Deployment Guide: Task Flow

This guide provides step-by-step instructions for deploying your redesigned **Task Flow** application.

---

## 1. Deploy the Backend on Railway

Railway is ideal for hosting your Node.js/Express server and PostgreSQL database.

### Step 1: Create a PostgreSQL Database on Railway
1. Log in to your [Railway Dashboard](https://railway.app/).
2. Click **+ New Project** -> **Provision PostgreSQL**. This will create a PostgreSQL instance.

### Step 2: Deploy the Backend Service
1. Click **+ New** (or **Add Service**) inside your project.
2. Select **GitHub Repo** (link your GitHub repository) and choose the `backend` folder as the root directory of the service.
   - *Note: If your repository contains both folders, you can set the **Root Directory** setting in your Railway service settings to `backend`.*
3. Railway will start building. It will automatically run `npm install` and trigger the `prisma generate` step.

### Step 3: Configure Environment Variables in Railway
Go to the **Variables** tab of your backend service on Railway and add the following variables:
- `PORT` = `5000` (or leave empty, Railway automatically sets it)
- `JWT_SECRET` = `[Choose a secure random string]`
- `FRONTEND_URL` = `https://your-frontend-domain.vercel.app` (your Vercel app URL once deployed)
- `DATABASE_URL` = `${{Postgres.DATABASE_URL}}` (or select the reference to the PostgreSQL database created in Step 1)
- `GOOGLE_CLIENT_ID` = `780927950547-ni8g7p09ujsdd5pp1farb6o3950baotr.apps.googleusercontent.com`

### Step 4: Run Migrations on the Database
To set up tables in your new PostgreSQL database on Railway:
1. In your local terminal, set your local `backend/.env`'s `DATABASE_URL` to your Railway PostgreSQL connection string (which you can copy from the Railway PostgreSQL service dashboard under *Connect* -> *Connection String*).
2. Run the migration command locally to push the schema to the cloud:
   ```bash
   npx prisma db push
   ```
3. (Optional) Run the seed command to create the default admin user:
   ```bash
   npm run seed
   ```

## 2. Deploy the Frontend on Vercel

Vercel is the easiest and most performant way to host React/Vite frontends.

### Step 1: Link your Repository to Vercel
1. Log in to your [Vercel Dashboard](https://vercel.com/).
2. Click **Add New** -> **Project**.
3. Import your GitHub repository.

### Step 2: Configure Project Settings
In the configuration screen before clicking Deploy:
1. **Framework Preset**: Select **Vite**.
2. **Root Directory**: Select `frontend`.
3. **Environment Variables**:
   - Add `VITE_API_URL` = `https://[your-backend-railway-url].railway.app/api` (this is the public URL generated by Railway for your backend service).
   - Add `VITE_GOOGLE_CLIENT_ID` = `780927950547-ni8g7p09ujsdd5pp1farb6o3950baotr.apps.googleusercontent.com`

### Step 3: Deploy
1. Click **Deploy**. Vercel will build and launch your application!

---

## 3. Running Locally (After Deployment Setup)
To run the project locally while connected to your production PostgreSQL database:
1. Set the database connection URL in `backend/.env`:
   ```env
   DATABASE_URL="postgresql://johndoe:password@db-host:port/dbname"
   ```
2. Start the backend:
   ```bash
   npm start
   ```
3. Start the frontend:
   ```bash
   npm run dev
   ```
